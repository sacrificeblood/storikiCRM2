# Доска связок — сервер

Полноценный бэкенд: свой Node.js/Express-сервер, база данных Postgres,
настоящая регистрация/логин (bcrypt + JWT-сессии в httpOnly-cookie) и простой
лог действий (кто и когда что менял). Всё это можно развернуть бесплатно.

## Из чего состоит

```
server.js      — Express: роуты авторизации + REST-хранилище для доски
db.js          — подключение к Postgres и создание таблиц при первом запуске
public/index.html — сама доска (фронтенд), без изменений в логике — только
                     точка входа теперь дергает ваш сервер, а не Firebase
.env.example   — какие переменные окружения нужны
```

Таблицы создаются автоматически при первом запуске сервера:
- `users` — email + bcrypt-хеш пароля
- `board_state` — данные доски (тот же JSON, что и раньше, теперь в вашей БД)
- `activity_log` — кто и когда логинился/сохранял/удалял данные

## Шаг 1. Бесплатная база Postgres

Любой из этих сервисов подходит (у всех есть бесплатный тариф без карты или с картой, но без списаний):

- **Neon** — neon.tech — самый простой вариант, бесплатный тариф с лихвой хватит для такой доски.
- **Supabase** — supabase.com — тоже подходит, база Postgres + веб-интерфейс для просмотра данных.

Создайте проект → скопируйте **Connection string** (обычно в разделе "Connect" / "Database URL"),
выглядит примерно так:
```
postgres://user:password@host.region.provider.tech/dbname?sslmode=require
```

## Шаг 2. Настроить .env

Скопируйте `.env.example` в `.env` и заполните:

```
DATABASE_URL=<строка подключения из шага 1>
JWT_SECRET=<длинная случайная строка>
INVITE_CODE=<придумайте код приглашения для регистрации команды>
```

Сгенерировать `JWT_SECRET` можно так:
```
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
```

`INVITE_CODE` — простой способ не пускать посторонних регистрироваться на вашем сайте.
Сообщите этот код только своей команде. Если оставить пустым — регистрация будет открыта всем.

## Шаг 3. Запуск локально (проверить, что всё работает)

```
npm install
npm start
```

Откройте http://localhost:3000 — увидите экран входа. Нажмите «Регистрация»,
введите email/пароль и код приглашения.

## Шаг 4. Бесплатный хостинг сервера

Подойдёт любой хостинг, поддерживающий Node.js-приложения. Проще всего — **Render.com**:

1. Залейте эту папку в репозиторий на GitHub (можно приватный).
2. На render.com → **New → Web Service** → подключите репозиторий.
3. Настройки: **Build command** `npm install`, **Start command** `npm start`.
4. В разделе **Environment** добавьте те же переменные, что в `.env`
   (`DATABASE_URL`, `JWT_SECRET`, `INVITE_CODE`, `NODE_ENV=production`).
5. Deploy. Через пару минут получите ссылку вида `https://ваш-сервис.onrender.com`.

Бесплатный тариф Render «засыпает» после ~15 минут без запросов и просыпается
секунд за 20-30 при следующем заходе — для внутреннего инструмента команды
это обычно не критично.

Альтернативы Render: Railway.app, Fly.io — процесс похожий (Git-репозиторий →
переменные окружения → деплой).

## Важно понимать про такую JWT-авторизацию

Логин делает настоящую сессию (подписанный токен в httpOnly-cookie, пароли
хранятся только как bcrypt-хеш — сам пароль нигде не хранится). Но токен
"живёт" 30 дней сам по себе: кнопка "Выйти" стирает cookie в браузере, но
если токен был кем-то перехвачен раньше — он останется рабочим до истечения
срока. Для внутреннего инструмента команды это обычно приемлемый компромисс;
если нужна возможность мгновенно отозвать доступ конкретному человеку —
это отдельная доработка (список отозванных токенов в базе), скажите — добавлю.
